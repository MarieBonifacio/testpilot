## Étape 4 — Streaming Ollama

### 4a — `proxy.js` : `ollamaStream`

Nouvelle fonction (juste après `ollamaRequest`) qui retourne `{ status, response: IncomingMessage }` au lieu de bufferiser le corps — permet de streamer les bytes SSE vers le client.

Ajouter à `module.exports` : `{ app, db, ollamaRequest, ollamaStream, authMiddleware }`.

### 4b — `routes/ollama.js` : route chat streaming

Remplacer le handler `POST /chat` pour supporter `req.body.stream`:
- `stream: false` → comportement actuel via `getOllamaRequest()` (conservé)
- `stream: true` → `getOllamaStream()` puis headers SSE (`Content-Type: text/event-stream`, `X-Accel-Buffering: no`, `Cache-Control: no-cache`), puis `ollamaRes.pipe(res)` + `req.on('close', () => ollamaRes.destroy())`

### 4c — `api.ts` : consommateur streaming frontend

Dans la branche `ollama` de `llmApi.call` :
- Envoyer `stream: true` dans le body
- Lire `response.body.getReader()` (ReadableStream — fetch-based SSE, pas EventSource car POST)
- Parser lignes `data: {...}` / `data: [DONE]`
- Accumuler `choices[0].delta.content` → retourner le texte final
